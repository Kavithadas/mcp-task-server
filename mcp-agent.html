📝 The Prompt Used With the Agent
"Please inspect the task schema at /mcp/schema/tasks. Then generate and insert 1000 diverse tasks with random statuses, titles, and due dates using the /mcp/tasks endpoint." What the agent does with this prompt: 1. STEP 1 — Calls GET /mcp/help to discover all available tools 2. STEP 2 — Calls GET /mcp/schema/tasks to understand: • Required fields: title, status • Valid statuses: TODO, IN_PROGRESS, DONE, CANCELLED, BLOCKED • Valid priorities: LOW, MEDIUM, HIGH, CRITICAL • Optional fields: description, priority, dueDate, assignedTo, category 3. STEP 3 — Generates 1000 diverse tasks: • Randomises status using weighted distribution • Uses 40+ realistic software engineering titles • Assigns to team members: alice, bob, carol, dave, eve, frank • Uses categories: Backend, Frontend, DevOps, QA, Security, Database • Sets due dates spread across the next 6 months • Adds realistic descriptions for ~70% of tasks 4. STEP 4 — POSTs all tasks to /mcp/tasks in batches 5. STEP 5 — Calls GET /mcp/tasks/summary to validate: • Confirms totalTasks matches what was inserted • Shows breakdown by status, priority, category, assignee
